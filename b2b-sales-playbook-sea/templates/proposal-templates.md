# Proposal Templates with AI Generation Prompts

**Last Updated**: 2026-02-07
**Purpose**: Ready-to-use proposal structures with AI prompts for each section

---

## How to Use These Templates

1. **Choose your template** based on deal type (SaaS, Services, Manufacturing)
2. **Fill in [BRACKETED] sections** with your company/prospect info
3. **Use AI prompts** to generate each section
4. **Customize** the output for your specific situation
5. **Always human-review pricing** (never let AI set price)

### Executive Summary

```
Write an Executive Summary for a SaaS proposal:

Prospect: [COMPANY NAME]
Industry: [INDUSTRY]
Size: [EMPLOYEES]
Key contact: [NAME], [TITLE]

Pain points from discovery:
- [PAIN 1 with quote]
- [PAIN 2 with quote]
- [PAIN 3 with quote]

Our solution: [1 SENTENCE DESCRIPTION]
Key value: [QUANTIFIED BENEFIT - e.g., "reduce onboarding time from 2 weeks to 3 days"]

Write 200-250 words, tone appropriate for [COUNTRY].

### The Challenge

**AI Prompt**:

```
Write a "Challenge" section (150-200 words):

Based on discovery notes:
[PASTE DISCOVERY NOTES]

Structure:
1. Current state (how they do things today)
2. Pain points (2-3 specific challenges with quotes)
3. Business impact (quantified)
4. Why now (urgency driver)

Tone: Empathetic, not accusatory. Frame as "challenges" not "failures".
```

**Template**:

```
THE CHALLENGE

Current Process:
[PROSPECT] currently [DESCRIBES CURRENT PROCESS - e.g., "manages customer onboarding through a combination of email, spreadsheets, and manual handoffs"].

This creates several challenges:

1. [PAIN POINT 1]
   "[QUOTE FROM DISCOVERY]" - [NAME], [TITLE]
   Impact: [QUANTIFIED - e.g., "3-4 days per customer, limiting throughput"]

2. [PAIN POINT 2]
   "[QUOTE]"
   Impact: [QUANTIFIED]

3. [PAIN POINT 3]
   "[QUOTE]"
   Impact: [QUANTIFIED]

Business Impact:
• [METRIC 1 - e.g., "30% customer drop-off during onboarding"]
• [METRIC 2 - e.g., "$15,000/month in ops team time"]
• [METRIC 3 - e.g., "Can't scale beyond 50 customers/month without hiring"]

Urgency:
With [PROSPECT]'s [GROWTH PLAN / DEADLINE / COMPETITIVE PRESSURE], solving this challenge is critical.

### Implementation Plan

**AI Prompt**:

```
Create implementation timeline (200-250 words):

Implementation duration: [X weeks]
Phases: [NUMBER]
Prospect context: [SIZE, COMPLEXITY, URGENCY]

For each phase:
- Duration
- Activities
- Deliverables
- Success criteria
- Teams involved

Tone: Realistic, shows you've thought through details.
```

**Template**:

```
IMPLEMENTATION PLAN

Total Duration: [X] weeks from contract signing to full deployment

## Phase 1: Discovery & Setup ([WEEKS])

**Activities**:
• Week [X]: Kickoff meeting, confirm requirements
• Week [X]: Technical discovery (APIs, data sources, integrations)
• Week [X]: Configure platform for [PROSPECT]'s workflows
• Week [X]: Set up sandbox environment

**Deliverables**:
• Technical integration plan
• Configured platform (sandbox)
• Team training materials

**Success Criteria**: Sandbox environment validated by [PROSPECT] team

**Teams**: Our implementation team (2 engineers, 1 PM) + [PROSPECT] [TEAMS]

## Phase 2: Pilot ([WEEKS])

**Activities**:
• Week [X]: Deploy to pilot group ([SIZE] users/customers)
• Week [X]: Monitor, gather feedback, iterate
• Week [X]: Training for [PROSPECT] team

**Deliverables**:
• Pilot deployment
• Trained [PROSPECT] team
• Pilot results report

**Success Criteria**: [METRIC] achieved in pilot (e.g., "onboarding time <3 days")

## Phase 3: Full Deployment ([WEEKS])

**Activities**:
• Week [X]: Roll out to all [USERS/CUSTOMERS]
• Week [X]: Migration of historical data (if needed)
• Week [X]: Go-live support

**Deliverables**:
• Full production deployment
• Documentation and runbooks
• Support handoff

**Success Criteria**: [METRICS] - e.g., "100+ users onboarded, system uptime >99%"

## Ongoing Support

• Dedicated Customer Success Manager
• 24/7 technical support
• Monthly business reviews
• Quarterly optimization sessions
```

## Proposal Formatting Checklist

Before sending, verify:

- [ ] All [BRACKETED] placeholders replaced with real info
- [ ] Prospect's company name spelled correctly throughout
- [ ] Contact names and titles accurate
- [ ] Numbers and metrics from discovery (not AI-generated guesses)
- [ ] Pricing accurate (human-reviewed, not AI)
- [ ] Currency appropriate for prospect's country
- [ ] Payment terms realistic for that market
- [ ] All links working (calendar, website, case studies)
- [ ] PDF formatted professionally (consistent fonts, spacing)
- [ ] File named properly (e.g., "Proposal_TokopediaOnboarding_2026-02-07.pdf")
